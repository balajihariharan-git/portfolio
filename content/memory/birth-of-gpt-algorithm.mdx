# The Full Story: How Google Invented AI But OpenAI Won the Race

> *One of the most dramatic and ironic stories in tech history*

---

## The Core Irony

Google invented:
- The Transformer (2017)
- BERT (2018)
- Word2Vec (2013)
- TensorFlow (2015)
- TPUs (custom AI chips)
- Deep Learning breakthroughs

Yet OpenAI — a startup — launched ChatGPT first and shocked the world, including Google.

## 1. Word2Vec (2013) — The Foundation

### What Problem Did It Solve?

Before Word2Vec, computers had NO idea that:

```
"King" and "Queen" are related
"Paris" and "France" are related
"Cat" and "Dog" are related

Words were just random IDs:
King  = 1
Queen = 2
Paris = 3
Cat   = 4
(No relationship between numbers)
```

### What Word2Vec Did

It created an algorithm that reads millions of sentences and learns:

```
"Words that appear together
 in similar sentences
 must be related"

Training sentences:
"The King ruled the kingdom"
"The Queen ruled the kingdom"
→ King and Queen appear in
  similar contexts
→ They must be related
→ Give them similar numbers

"Paris is in France"
"London is in England"
→ Paris/London are similar (cities)
→ France/England are similar (countries)
```

### The Algorithm — Super Simple Idea

```
STEP 1:
Take a sentence:
"The cat sat on the mat"

STEP 2:
For each word, look at
surrounding words (context window):

"sat" appears near → "cat", "on", "the"
"cat" appears near → "the", "sat"
"mat" appears near → "the", "on"

STEP 3:
Train a tiny neural network:
Input: word "sat"
Predict: surrounding words
"cat", "on", "the"

STEP 4:
The hidden layer numbers
that make this prediction work
= the embedding (meaning vector)

Do this for all words
in millions of sentences
→ Every word gets
  meaningful numbers
```

### What It Produced — The Famous Result

```
King   = [0.9, 0.8, 0.1, 0.7, 0.2 ...]
Queen  = [0.9, 0.8, 0.1, 0.3, 0.2 ...]
Man    = [0.5, 0.2, 0.1, 0.7, 0.6 ...]
Woman  = [0.5, 0.2, 0.1, 0.3, 0.6 ...]

King - Man + Woman = Queen

Paris  = [0.2, 0.8, 0.5, 0.1 ...]
France = [0.2, 0.7, 0.4, 0.9 ...]
London = [0.2, 0.9, 0.5, 0.1 ...]
England= [0.2, 0.8, 0.4, 0.9 ...]

Paris - France + England = London

MEANING encoded in MATH
```

### What It Is Made of Technically

```
Just ONE small neural network:

Input Layer    Hidden Layer    Output Layer
(one word)     (300 numbers)   (predict nearby words)

[King]    →    [0.9, 0.8...]  →   [Queen, throne, royal]
[Paris]   →    [0.2, 0.8...]  →   [France, city, Eiffel]

The hidden layer numbers
= the word embedding
= the meaning of the word
```

Word2Vec is made of just:
- **One neural network**
- **One simple task** (predict nearby words)
- **Basic math** (matrix multiplication)

## 2. TensorFlow (2015) — The Construction Tools

### What Problem Did It Solve?

Before TensorFlow:

```
Every researcher wrote their own code
from scratch to do matrix math on GPUs

Like:
Every builder makes their own
hammer, saw, drill from raw metal
before they can build anything

Took months just to set up
basic training infrastructure
```

### What TensorFlow IS

It is a **software library** — a collection of pre-written code functions.

```python
# WITHOUT TensorFlow
# You write 500 lines of complex GPU math code:
import cuda
kernel = cuda.compile("""
  __global__ void matmul(
    float *A, float *B, float *C, int N) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    float sum = 0;
    for (int k = 0; k < N; k++)
      sum += A[row*N+k] * B[k*N+col];
    C[row*N+col] = sum;
  }
""")
# ... 500 more lines

# WITH TensorFlow
# Same thing in 1 line:
result = tf.matmul(A, B)
```

### What It Is Made of — 3 Layers

```
LAYER 1 — C++ Core
Low level GPU operations
Written in C++ for maximum speed
Talks directly to NVIDIA GPU drivers

LAYER 2 — Python API
Makes C++ operations accessible
via simple Python commands:
tf.add(), tf.multiply(), tf.matmul()

LAYER 3 — High Level Tools
Keras        — build neural networks in a few lines
Estimators   — standard training loops
TensorBoard  — visualize training progress

You only ever touch Layer 2 and 3.
Layer 1 runs automatically.
```

### The Key Concept — What is a "Tensor"?

```
Tensor = just a multi-dimensional array of numbers

0D tensor = single number:      42
1D tensor = list:               [1, 2, 3, 4]
2D tensor = table:              [[1,2],[3,4]]
3D tensor = cube of numbers
4D tensor = how images are stored

"Flow" = how data flows through the neural network

TensorFlow = how tensors (numbers)
             flow through the network
```

## 3. Transformer (2017) — The Blueprint

### What Problem Did It Solve?

Before Transformer, the best models were called **RNN** (Recurrent Neural Networks):

```
RNN reads text sequentially — word by word:

"The cat sat on the mat
 because it was tired"

RNN reads:
"The" → remembers
"cat" → remembers
"sat" → remembers
...
"it"  → tries to remember what "it" refers to
        BUT by now memory of "cat" has faded!

RNN FORGETS early context
by the time it reaches end of long sentences
```

### What Transformer Did

```
Transformer reads ALL words at the SAME TIME
and lets every word look at every other word:

"The cat sat on the mat because IT was tired"

When processing "IT":
IT looks at ALL other words simultaneously:
IT <-> "The"     (low attention)
IT <-> "cat"     (HIGH attention)
IT <-> "sat"     (medium attention)
IT <-> "on"      (low attention)
IT <-> "mat"     (low attention)
IT <-> "because" (low attention)
IT <-> "was"     (medium attention)
IT <-> "tired"   (medium attention)

Model learns: IT = cat
```

### What It Is Made of — The 6 Parts

```
1. INPUT EMBEDDING
   Convert tokens to vectors
   (Word2Vec idea but better, learned during training)

2. POSITIONAL ENCODING
   Tell the model word order
   "cat" is word 2, "sat" is word 3
   (since all words are read simultaneously,
    model needs to know their positions)

3. SELF ATTENTION (the magic)
   Every word looks at every other word
   and scores: "How much should I pay
   attention to you to understand myself?"

   Uses 3 matrices:
   Q = Query  (what am I looking for?)
   K = Key    (what do I contain?)
   V = Value  (what do I give if selected?)

   Attention = softmax(QK / sqrt(d)) x V

   This one formula is the heart of all LLMs.

4. FEED FORWARD NETWORK
   Process the attended information
   through a small neural network

5. REPEAT 6-96 TIMES (layers)
   Each layer understands deeper relationships

6. OUTPUT
   Predict next token
```

### The Attention Formula in Plain English

```
Every word asks 3 questions:

QUERY (Q): "I am looking for..."
KEY   (K): "I contain..."
VALUE (V): "If you pick me, take this..."

Example with "IT":

IT's Query:  "I am looking for what I refer to"
cat's Key:   "I am a living thing that can be tired"
cat's Value: "cat"

→ IT's Query matches cat's Key
→ IT attends strongly to cat
→ IT receives cat's Value
→ Model understands: IT = cat
```

## 4. BERT (2018) — First Completed House

### What Problem Did It Solve?

GPT-1 and early Transformers read text **left to right only**:

```
"The bank can guarantee deposits
 will eventually cover future bank losses"

Reading left to right only:
"bank" → is this a river bank or financial bank?

Model sees what came BEFORE "bank"
but not what comes AFTER.

If it read BOTH directions:
"bank" + "deposits" + "losses"
→ Obviously financial bank
```

### What BERT Did

```
BERT = Bidirectional Encoder Representations
       from Transformers

Reads text in BOTH directions simultaneously:

<- "losses cover future bank deposits"
-> "The bank can guarantee deposits"

Understands full context from both sides
of every word
```

### How BERT Was Trained — Two Clever Tasks

```
TASK 1 — Masked Language Model (MLM):

Take sentence:
"The cat sat on the mat"

Randomly hide 15% of words:
"The [MASK] sat on the [MASK]"

Train model to predict:
[MASK] 1 = "cat"
[MASK] 2 = "mat"

To predict correctly, model MUST understand
full context from both directions

TASK 2 — Next Sentence Prediction (NSP):

Show model two sentences:
A: "The cat sat on the mat"
B: "It was feeling sleepy"
→ Does B follow A? YES

A: "The cat sat on the mat"
B: "Pizza is delicious"
→ Does B follow A? NO

Model learns document-level understanding,
not just individual words
```

### What BERT Is Made of

| Version | Transformer Layers | Hidden Size | Attention Heads | Parameters |
|---------|-------------------|-------------|-----------------|------------|
| BERT Base | 12 | 768 | 12 | 110 million |
| BERT Large | 24 | 1024 | 16 | 340 million |
| GPT-4 (for comparison) | 96+ | 12,288 | 96 | 1.7 **trillion** |

## How Raw Data Gets Cleaned — The Pipeline

Every major AI lab runs a cleaning pipeline before training:

```
RAW COMMON CRAWL
(100 TB messy HTML)
        |
STEP 1 — HTML EXTRACTION
Remove all tags, keep only text:
"The capital of France is Paris.
 Einstein developed the theory of relativity."

        |
STEP 2 — LANGUAGE DETECTION
Keep only desired languages
(English, or multilingual as needed)
Remove: gibberish, code-switched text

        |
STEP 3 — QUALITY FILTERING
Remove pages with:
+-- Too many special characters
+-- Too short (< 100 words)
+-- Too many repeated lines (spam)
+-- High perplexity (gibberish)
+-- Known spam domains

        |
STEP 4 — DEDUPLICATION
Same article appears on 10,000 news sites
Keep only ONE copy
Reduces data by 30-70%

        |
STEP 5 — TOXICITY FILTERING
Remove pages containing:
+-- Hate speech
+-- Violence
+-- Adult content
+-- Dangerous information
+-- Extreme bias

        |
STEP 6 — QUALITY SCORING
Some labs use a "quality classifier"
trained on high-quality human-written text
(Wikipedia, books, academic papers)
to score and filter low quality pages

        |
CLEAN TRAINING DATA
Ready to train on
```

## The Pre-Training Process: Step by Step

Now that you have the clean data, here is exactly what happens:

### Step 1 — Tokenization

```
Clean text:
"The quick brown fox jumps over the lazy dog"

Tokenized via SentencePiece / tiktoken:

Tokens (IDs):
[464, 2068, 7586, 21831, 18045, 625, 262, 16931, 3290]

The entire dataset (trillions of words)
becomes a single MASSIVE sequence of numbers:

[464, 2068, 7586, 21831, 18045, 625, 262, 16931, 3290,
 1212, 318, 262, 3139, 286, 4881, 13, 48302, 373,
 257, 4925, 7309, 2492, 290, 257, 3199, 13, ...]

(Hundreds of billions of numbers in sequence)
```

### Step 2 — Create Training Examples

```
Take a sliding window of 2048 tokens (GPT-3) or
4096/8192/128K tokens (newer models):

Window 1:
INPUT:  [The, quick, brown, fox, jumps, over, the, lazy]
TARGET: [quick, brown, fox, jumps, over, the, lazy, dog]

Window 2 (slide by 1):
INPUT:  [quick, brown, fox, jumps, over, the, lazy, dog]
TARGET: [brown, fox, jumps, over, the, lazy, dog, Paris]

The model's TASK:
Given these tokens → predict the next token
This single task, repeated trillions of times,
is ALL of pre-training.
```

### Step 3 — The Training Loop

```
FOR each batch of training examples:

  1. FORWARD PASS
     Feed token sequence into Transformer
     Model predicts probability of next token
     for EVERY position simultaneously

     Example:
     Input:  "The capital of France is ___"
     Output: {"Paris": 0.73, "city": 0.08,
              "beautiful": 0.05, "known": 0.04 ...}

  2. CALCULATE LOSS (Cross-Entropy Loss)

     Loss = -log(P(correct_token))

     If correct = "Paris" (probability 0.73):
     Loss = -log(0.73) = 0.315  — small, good

     If model said "banana" (probability 0.001):
     Loss = -log(0.001) = 6.908  — huge, bad

     Average loss over entire batch = batch loss

  3. BACKWARD PASS (Backpropagation)
     Calculate gradient:
     "Which direction should each weight move
      to reduce the loss?"

     dLoss/dW for every weight W in model
     This is billions of calculations
     Done automatically by PyTorch/JAX

  4. UPDATE WEIGHTS (Gradient Descent)

     W_new = W_old - learning_rate x gradient

     Learning rate (e.g. 0.0001):
     How big a step to take
     Too large = unstable training
     Too small = too slow

     AdamW optimizer (used by most LLMs):
     Adaptive learning rate per weight
     Better than basic gradient descent

  5. REPEAT
     For the entire dataset
     Multiple times (epochs)
     Billions of parameter updates total
```

### Step 4 — What Gets Learned Automatically

Nobody programs these in. They EMERGE from predicting next tokens:

```
After seeing "Paris is the capital of France"
and "London is the capital of England"
and "Tokyo is the capital of Japan"
millions of times...

Model learns:
[city] is the capital of [country]
pattern exists everywhere

After seeing code millions of times:
Model learns syntax, indentation, logic

After seeing maths millions of times:
Model learns arithmetic, algebra patterns

After seeing conversations:
Model learns dialogue patterns

After seeing reasoning:
Model learns chain-of-thought patterns

NOTHING was explicitly programmed.
Everything emerged from next-token prediction
on enough diverse data.
```

## The Legal Battle — Did They Have the Right?

This is the most controversial aspect of LLM training data.

### The Legal Arguments

```
PRO (Companies argue):
+-- Public web content = publicly accessible
+-- Web scraping is legal (hiQ vs LinkedIn, 2022)
+-- Training is not copying (transformative use)
+-- Fair use doctrine protects research use
+-- Robots.txt was not always followed but
    was not legally binding

AGAINST (Authors/publishers argue):
+-- Copyright is not public domain
+-- No consent was given
+-- Revenue lost (AI replaces reading originals)
+-- Style and voice can be extracted
+-- Scale makes it different from research
```

### Major Lawsuits Filed Against OpenAI

```
December 2023 — New York Times vs OpenAI
"OpenAI used millions of NYT articles
 to train GPT without permission or payment"
Status: Ongoing — largest AI copyright case

2023 — Authors Guild vs OpenAI
Signed by: John Grisham, George R.R. Martin,
           Jodi Picoult, and 8,000+ authors
"Used our books without consent or compensation"
Status: Ongoing

2023 — Sarah Silverman et al vs OpenAI + Meta
"Used copyrighted books for training"
Status: Partially dismissed, partially ongoing

Getty Images vs Stability AI
"Used 12 million copyrighted images"
Status: Ongoing

Music publishers vs Anthropic (Claude)
"Used copyrighted song lyrics"
Status: Ongoing
```

### The "Distillation" Controversy

```
DeepSeek admitted in their paper:
"Our pre-training dataset relied on crawled
 web pages that contained a significant number
 of OpenAI-model-generated answers"

Translation: They used ChatGPT outputs
             to train their model

OpenAI's ToS explicitly FORBIDS this.

But OpenAI itself used:
Reddit posts, books, news articles
WITHOUT those authors' explicit permission.

The irony: Everyone is using everyone else's data.
```

### How the Industry Is Responding

```
Moving toward licensed data:
+-- OpenAI — paying AP, Axel Springer, others
+-- Google — licensing news content
+-- Apple  — licensing news for Apple Intelligence
+-- Meta   — licensing from some publishers

New data sources emerging:
+-- Synthetic data (AI generates training data)
+-- Human-written data marketplaces
+-- Consent-based data collection
+-- Federated learning (train on device, not server)
```

## One Paragraph Summary

> Common Crawl, a free nonprofit archive of the public internet founded in 2007, is the backbone of almost every major LLM ever built. GPT-1 used books, GPT-2 used Reddit links, GPT-3 used 60% Common Crawl data totalling 499 billion tokens, and GPT-4 used an estimated 10-13 trillion tokens from similar sources. The data was legally scraped from public websites — but without author consent — triggering massive lawsuits from the New York Times, thousands of authors, and Getty Images that are still ongoing. The irony is that Google, which invented the Transformer and had the best AI, also had access to the same data but refused to build products with it. OpenAI took the same publicly available data, trained on it aggressively, and changed the world while Google watched.
